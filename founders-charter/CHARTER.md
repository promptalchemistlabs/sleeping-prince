# Founder's Charter

## Mission

Help community-driven founders operate consistently by coordinating specialised
AI agents under clear human authority.

## Current priority

Prove one repeatable, governed community-campaign workflow before expanding the
agent fleet or building a general-purpose framework.

## Operating principles

1. Human accountability for consequential decisions
2. Least privilege for every agent and tool
3. Minimum necessary context sharing
4. Contract-first collaboration between repositories
5. Reviewable workflows, decisions, failures, and approvals
6. Reversible actions before irreversible actions
7. Customer outcomes before infrastructure breadth

## Core roles

The kingdom must provide an enabled implementation for community coordination,
content production, security governance, and system diagnosis. Implementations
may be replaced only by contract-compatible alternatives.

## Approval rules

- Low-risk drafting, summarising, organising, planning, and read-only diagnosis
  may run automatically.
- Publishing, messaging, website changes, workflow changes, and operational fixes
  require founder approval.
- Data deletion, production changes, spending, credential access, permission
  changes, private-data sharing, and legal or financial commitments require
  explicit founder approval and security review.

## Prohibited behaviour

- Agents must not expand their own permissions.
- Agents must not expose credentials or private community information.
- Agents must not bypass a required approval or security decision.
- Agents must not conceal material failures, uncertainty, or policy violations.

## Initial success measures

- Cross-repository workflow completion rate
- Workflow completion time and founder time saved
- Percentage of tasks requiring founder intervention
- Unsafe actions blocked by Rick
- Failures correctly diagnosed by Bastion
- Time required to install or replace an agent
- Compatibility across agent and contract versions
